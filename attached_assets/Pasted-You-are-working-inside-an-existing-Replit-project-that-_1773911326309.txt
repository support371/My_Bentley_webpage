Mission

1. receive Bentley webhook events,
2. persist them,
3. correlate them by tenant / iTwin / iModel,
4. enrich them with Bentley API data,
5. visualize them in a responsive dashboard,
6. trigger alerts and automations,
7. expose admin tooling for Bentley configuration,
8. support pilot-grade customer use.

Do not stop at a cosmetic dashboard refresh. Complete the full platform.

Product outcome

4) Webhook ingestion
Upgrade /webhook into a proper ingestion pipeline.
Requirements:
- Verify signature in production mode
- Reject invalid signatures
- Support dev mode override through config only
- Validate payload shape
- Capture raw request body
- Capture headers
- Deduplicate on Bentley message id when present
- Process asynchronously using background tasks or a queue abstraction
- Return safe status payloads
- Log all failures to webhook_deliveries and audit_logs

Also build:
- webhook replay tool
- dead-letter handling for failed processing
- retry endpoint for admins
